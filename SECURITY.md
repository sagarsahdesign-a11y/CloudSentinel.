# Security Policy

## Supported Versions

Only the latest major version of CloudSentinel is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

---

## Reporting a Vulnerability

As a security scanning tool, we take security vulnerabilities very seriously. If you find a security vulnerability in CloudSentinel, please do not disclose it publicly or file a public GitHub issue.

Please report vulnerabilities by emailing **security@cloudsentinel.example.org** (or your designated security address).

Please include:
- A detailed description of the vulnerability.
- Steps to reproduce (proof of concept).
- Potential impact and mitigation.

We will acknowledge receipt of your report within 48 hours and work with you to coordinate a security patch and release.
